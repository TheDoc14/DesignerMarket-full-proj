2. תכנות מונחה עצמים (OOP)
א. מחלקה (Class) ואובייקט (Object):
מחלקה: תבנית ליצירת אובייקטים. מכילה תכונות (שדות) ומתודות (פונקציות).

אובייקט: מופע של מחלקה (instance).

java
Copy
public class Student {
    private String name; // תכונה פרטית
    private int id;

    // בנאי (Constructor)
    public Student(String name, int id) {
        this.name = name;
        this.id = id;
    }

    // מתודת גישה (Getter)
    public String getName() { return name; }

    // מתודת הדפסה
    public void printInfo() {
        System.out.println("Name: " + name + ", ID: " + id);
    }
}

// יצירת אובייקט
Student s1 = new Student("אדם", 123);
s1.printInfo(); // פלט: Name: אדם, ID: 123
ב. עקרונות OOP:
אינקפסולציה (Encapsulation):
הסתרת נתונים וחשיפת ממשק מבוקר באמצעות private ו-public.

java
Copy
public class BankAccount {
    private double balance; // תכונה פרטית
    public void deposit(double amount) { balance += amount; }
}
ירושה (Inheritance):
מחלקה יורשת תכונות ומתודות ממחלקה אחרת.

java
Copy
public class Animal {
    public void sound() { System.out.println("Animal sound"); }
}

public class Dog extends Animal {
    @Override
    public void sound() { System.out.println("Woof!"); }
}
פולימורפיזם (Polymorphism):
יכולת של אובייקט להתנהג בדרכים שונות בהתאם להקשר.

Copy

myAnimal.sound(); // Woof! (למרות שהטיפוס הוא Animal)
דריסה (Overriding):
החלפת מימוש של מתודה במחלקה יורשת. חובה להשתמש ב-@Override.

java
Copy
public class Vehicle {
    public void move() { System.out.println("Vehicle is moving"); }
}

public class Car extends Vehicle {
    @Override
    public void move() { System.out.println("Car is moving fast"); }
}
העמסה (Overloading):
הגדרת מספר מתודות עם אותו שם אך פרמטרים שונים.

java
Copy
public class Calculator {
    public int add(int a, int b) { return a + b; }
    public double add(double a, double b) { return a + b; }
}
מחלקה מופשטית (Abstract Class):
מחלקה עם לפחות מתודה אבסטרקטית אחת. לא ניתן ליצור ממנה אובייקטים ישירים.

java
Copy
public abstract class Shape {
    public abstract double area(); // מתודה מופשטית (חובה לממש במחלקה יורשת)
}

public class Circle extends Shape {
    private double radius;
    @Override
    public double area() { return Math.PI * radius * radius; }
}

3. מחסנית (Stack) – LIFO
א. פעולות עיקריות:
push() – הכנסת איבר.

pop() – הוצאת האיבר האחרון.

peek() – בדיקת האיבר העליון.

isEmpty() – בדיקת ריקות.

ב. העתקת מחסנית לפני שימוש:
חשיבות: פעולות כמו pop() משנות את המחסנית המקורית.

דוגמה:

java
Copy
Stack<Integer> original = new Stack<>();
original.push(10);
original.push(20);

// יצירת עותק באמצעות clone()
Stack<Integer> copy = (Stack<Integer>) original.clone();

// יצירת עותק ידני
Stack<Integer> manualCopy = new Stack<>();
for (int num : original) {
    manualCopy.push(num);
}

// שימוש בעותק
while (!copy.isEmpty()) {
    System.out.println(copy.pop()); // 20, 10
}

// המחסנית המקורית נותרה ללא שינוי
System.out.println(original.size()); // 2
4. תור (Queue) – FIFO
א. פעולות עיקריות:
add() – הכנסת איבר.

poll() – הוצאת האיבר הראשון.

peek() – בדיקת האיבר הראשון.

isEmpty() – בדיקת ריקות.

ב. העתקת תור לפני שימוש:
חשיבות: פעולות כמו poll() משנות את התור המקורי.

דוגמה:

java
Copy
Queue<Integer> originalQueue = new LinkedList<>();
originalQueue.add(10);
originalQueue.add(20);

// יצירת עותק
Queue<Integer> copyQueue = new LinkedList<>(originalQueue);

// שימוש בעותק
while (!copyQueue.isEmpty()) {
    System.out.println(copyQueue.poll()); // 10, 20
}

// התור המקורי נותר ללא שינוי
System.out.println(originalQueue.size()); // 2
5. השוואה בין מחסנית לתור
מאפיין	מחסנית (Stack)	תור (Queue)
עיקרון	LIFO (אחרון נכנס ראשון יצא)	FIFO (ראשון נכנס ראשון יצא)
סיבוכיות זמן	O(1) לכל הפעולות	O(1) לכל הפעולות
שימוש נפוץ	בדיקת סוגריים, רקורסיה	BFS, ניהול תורים
דוגמאות קוד	push(), pop()	add(), poll()
6. דוגמאות קוד לשימושים נפוצים
א. בדיקת תקינות סוגריים עם מחסנית:
java
Copy
public static boolean checkBrackets(String expression) {
    Stack<Character> stack = new Stack<>();
    for (char c : expression.toCharArray()) {
        if (c == '(' || c == '{' || c == '[') {
            stack.push(c);
        } else if (c == ')' || c == '}' || c == ']') {
            if (stack.isEmpty()) return false;
            char top = stack.pop();
            if ((c == ')' && top != '(') || 
                (c == '}' && top != '{') || 
                (c == ']' && top != '[')) {
                return false;
            }
        }
    }
    return stack.isEmpty();
}
ב. המרת תור למספר (כמו בבגרות 2019):
java
Copy
public static int toNumber(Queue<Integer> q) {
    Stack<Integer> stack = new Stack<>();
    while (!q.isEmpty()) {
        stack.push(q.poll()); // העברת האיברים למחסנית
    }
    int num = 0;
    while (!stack.isEmpty()) {
        num = num * 10 + stack.pop(); // בניית המספר
    }
    return num;
}
7. קיצורים וטיפים למבחן
קריאת קוד:

התייחס ללולאות ולפעולות על מבני נתונים (למשל: push, pop, add, poll).

זכור ש-pop() ו-poll() משנים את המבנה.

כתיבת קוד:

צור עותק של מחסנית/תור לפני שינויים אם נדרש לשמור על הנתונים המקוריים.

השתמש ב-isEmpty() לפני כל פעולת הוצאה.

ניהול זמן:

פתור תרגילים עם מחסנית ותור ראשון – הם לרוב פשוטים יותר.

בדוק תנאי קצה (למשל: מחסנית/תור ריקים).
